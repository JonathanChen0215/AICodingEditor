1. Product scope

AICodingEditor is a multi-language code editor and workbench. It combines code editing, syntax highlighting, Workbench and file browsing, Embedded Console execution, Python uv environments, SQL workflows, AI learning projects, and program-run data reports.

Languages
Java, C, C++, C#, Python, JavaScript, PHP, Rust, Arduino, SQL.
Databases
MS SQL Server, SQLite, MySQL CLI workflow.
AI
Machine Learning, Deep Learning, Neural Networks, computer vision and NLP workflows.
Run reports
Variables, Program stdin/stdout, tables, figures, XLSX reports and PNG charts.

2-1. AI Assistant layout and safety flow

The right-side AI Assistant panel contains the AI action strip, conversation area, Prompt / AI command window, attachment strip, and the bottom-left + attach menu.

AI Assistant: choose Chat, Explain, Fix Error, Generate Code, or Review Code from the top icon strip; click ▶ or press Ctrl+Enter to send. The bottom-left + menu attaches files, images, folders, or uv projects; files can also be dragged from the left tree into the Prompt area.

  • AI Provider Settings: Provider, default AI action, selected-code/current-file/Console Error context, Full Project Scan, Auto Run, Auto Install Package, and API-key source are configured here.
  • Safe defaults: AI never auto-runs code, never auto-installs packages, and never modifies files automatically.
  • Patch and new files: Preview / Diff is required before the user confirms Apply Patch or Create File.

3. Quick start

  1. Keep AICodingEditor.exe and AICodingEditor.jar in the same folder.
  2. Open a saved source file, then choose Compile Current or Run Current.
  3. For interactive programs, type input in the lower Program stdin field and press Send / Enter.
  4. For run reporting, enable VARIABLES / CHARTS → Enable Live Monitor, choose variables, then run the program.
Compile only checks/builds code. Run executes the program. Each actual program run creates its report folder beside the source file under variable.

Important limitation: AICodingEditor does not read private in-memory variables from arbitrary Java/C/C++/Python binaries. That would require language-specific debugging/instrumentation and is unsafe to do automatically. Output the values you want retained using AICE_VAR or name=value. This portable protocol works across all supported languages.

6. Live Variable Monitor and waveform

  1. Click the ⌁ Enable Live Monitor toolbar icon only when you need live telemetry. It is disabled and hidden by default.
  2. Click Select Variables.... Select detected names or type custom variable names separated by commas. Leaving the selection empty monitors all recognised output variables.
  3. Run the program. The bottom Variable Monitor tab appears only after Live Monitor is enabled; it is hidden by default on startup.
  4. The table shows current value, sample count, timestamp and source. Select a numeric row, or choose a name in the waveform selector, to view its live trend.
  5. After the program exits, click Open variable Reports to open the generated XLSX / table / figure folder.

The live monitor records samples while the process is running. It retains the complete numeric sample history for the current run. The live chart decimates only drawing pixels for exceptionally large runs; CSV and XLSX retain every row.

V0.1.35: uv startup check and Python package recovery

When AICodingEditor starts, it verifies the configured uv command. If uv is not available, it automatically tries:

python -m pip install uv

If the normal installation location is not writable, the editor retries with --user. The operation is shown in the Embedded Console. A failed bootstrap leaves the editor usable and displays a diagnostic; select the Python interpreter and uv path under Settings → Preference → General.

Missing third-party site-package while running Python

When a Python program exits with ModuleNotFoundError: No module named 'package', the editor shows a confirmation dialog. It proposes a PyPI name (for example sklearn → scikit-learn and cv2 → opencv-python). Select OK to install the package in the current Python / uv environment; after a successful install, the editor automatically runs the same file again.

# Example output that triggers recovery
ModuleNotFoundError: No module named 'humanize'

The distribution includes Examples/Python/AutoInstall_Humanize_Demo.py for this flow.

Python standard library reminder

Standard-library modules are not installed through PyPI. When the editor recognises a common standard-library NameError, such as name 'json' is not defined, it displays an import reminder instead of offering pip installation:

import json

Use Examples/Python/StandardLibrary_Import_Reminder_Demo.py to test this behavior. That sample intentionally comments out import json.

Safety: AICodingEditor asks before installing a third-party package after a Python run fails. Internet access and a working pip are required. The editor performs one automatic retry per manual run to avoid an installation loop.

V0.1.35: EXE-sibling toolchains and Jupyter Notebook

Toolchain folder layout beside AICodingEditor.exe

On first launch, the default Common toolchain root is the folder containing AICodingEditor.exe and AICodingEditor.jar. Place the existing third-party tool folders as siblings of the launcher:

Windows_Executable/
  AICodingEditor.exe
  AICodingEditor.jar
  JDK-26.0.1/        (Java tools)
  Dev-Cpp/           (MinGW gcc/g++)
  ArduinoCLI/        (arduino-cli.exe)
  SQL_Driver/        (SQLite / MS SQL driver files)
  Manuals/
  Examples/

AICodingEditor prepends existing sibling tool folders to the PATH of every compiler, runner, Python, Streamlit, Jupyter, and Embedded Console child process. It does not change the Windows system-wide or user-wide PATH registry setting and does not require administrator rights. Change the root only when your tool folders are elsewhere: Settings → Preference → General → Languages & Toolchains.

Editable .ipynb Notebook files

Open a Jupyter .ipynb file normally. AICodingEditor displays it in cell-text mode:

# %%
# Python code cell
print("Hello Jupyter")

# %% [markdown]
# Markdown cell title and notes

Save writes standard .ipynb JSON. Code-cell outputs are cleared when cell text is edited, preventing stale results from being kept after source changes.

Run Current Notebook

  1. Open and save an .ipynb file.
  2. Click Jupyter Run, or choose Run → Jupyter: Run Current Notebook (.ipynb).
  3. The selected Python / uv environment is checked for jupyter, nbconvert, and ipykernel.
  4. When missing, confirm the package-install request. AICodingEditor uses uv pip install --python <interpreter> jupyter nbconvert ipykernel when uv is available; otherwise it uses python -m pip install.
  5. AICodingEditor registers the selected Python environment as a Jupyter kernel, then executes all code cells with jupyter nbconvert --to notebook --execute --inplace. Outputs are saved into the same notebook file. Use Stop to terminate notebook execution.

The included Examples/Jupyter/SinCos_Live_Notebook.ipynb is a no-third-party-package starting example.

V0.1.36: Jupyter integer-counter repair

Before a .ipynb file is executed, AICodingEditor normalizes the notebook fields that nbformat requires to be JSON integers: nbformat, nbformat_minor, every code cell’s execution_count, and every execute_result.execution_count.

Automatic repair: an integral decimal such as "execution_count": 1.0 is rewritten as "execution_count": 1. An invalid/non-integral execution counter is changed to null, which is schema-valid and avoids inventing an execution order. Normal floating-point values in DataFrames, plots, image outputs, and unrelated metadata are not changed.

When any repair is needed, the editor first creates a sibling backup:

MyNotebook.aicodingeditor-backup-YYYYMMDD_HHMMSS_SSS.ipynb

This removes validation failures such as 1.0 is not of type 'integer', 'null' before nbconvert starts the kernel.

V0.2.37-02 — OpenAI + Gemini Environment-Variable API Keys

V0.2.37-02 retains the dockable right-side AI Assistant and adds per-provider API-key sources. For a new installation, Environment variable (recommended) is the default. Existing V0.2.37 encrypted local keys remain compatible and retain their source setting until changed.

  1. Fully close AICodingEditor. In Windows, open Edit the system environment variables → Environment Variables → User variables → New. Create OPENAI_API_KEY for OpenAI and/or GEMINI_API_KEY for Gemini. Start a new AICodingEditor process afterwards.
  2. Open AI → AI Provider Settings. For each provider choose Environment variable (recommended), retain the default variable name, confirm the detected status, and save. Gemini also accepts GOOGLE_API_KEY as a fallback when the Gemini field remains at its default GEMINI_API_KEY.
  3. Environment-variable values are read only when a request is sent through System.getenv(). They are never displayed, copied into the key box, or written to Java Preferences. Clear Locally Stored Keys never changes Windows environment variables.
  4. Use Chat, Explain Code, Fix Error, Generate Code, or Review Code. Selection, current file, and Console Error are enabled by default. Full Project Scan remains disabled by default and must be explicitly selected.
  5. For a code replacement, the model must return an AICodingEditor patch. Click Patch Diff Preview. Review the unified diff and side-by-side content, then click Apply Patch and confirm once more.
  6. After an AI reply returns one complete fenced code block or a complete patch, click Create New File... in the AI panel or choose AI → Create New File from AI Result.... The Save dialog starts in the active file folder/current working folder. Select a name and path, inspect the preview, and confirm before a file is created. If the target already exists, inspect the Diff and confirm overwrite separately. A target named .ipynb, or a Jupyter/Notebook request, is written as a standard Jupyter notebook and opened in cell-text mode.

Security: environment variables keep a key out of source files and AICodingEditor's Java Preferences, but they are not an enterprise secret vault; software running as the same Windows account may access them. Never commit a key to Git. The package includes SET_API_KEYS_WINDOWS.txt and Windows_Executable/Verify_AI_Environment_Variables.bat; the verification script reports only SET / NOT SET.

Safety: V0.2.37-02 never lets AI automatically run programs or install packages. A patch only changes the editor buffer after preview and confirmation. An AI-generated file is never written automatically: select a path, inspect the preview/Diff, then confirm creation or overwrite.

V0.2.37-07: Delete and drag-and-drop for the left Workbench/File Browser

The two left navigation trees now support Windows Explorer file drops, the Delete key, and context menus without adding large buttons. This preserves usable space for the Workbench, File Browser, editor, and AI panes.

Upper Workbench / Project

  • Drop a folder from Windows Explorer onto Workbench: after confirmation, it is registered as a project root.
  • Drop a file onto Workbench: after confirmation, its parent folder is registered as a project root and the file is opened.
  • Select a project root and press Delete: it is removed from the Workbench list only; no disk files are deleted.
  • Select a file or folder inside a project and press Delete: a permanent-delete warning is shown; the item does not go to the Recycle Bin.

Lower File Path / Browser

  • Drop files or folders from Windows Explorer onto a destination folder row: after confirmation, copies are imported into that folder. Source items are not moved or deleted; matching names are overwritten only after confirmation.
  • Select an item and press Delete: the file or folder is permanently deleted after confirmation. If an affected editor tab has unsaved changes, the editor first warns that the changes will be discarded.
  • The context menu provides Open, Show in File Explorer, Refresh, and, where applicable, Remove from Workbench or Permanently Delete.
Important: Permanent Delete does not move an item to the Windows Recycle Bin. Use it only when recovery is not needed. To remove a project safely, select the project root in the upper Workbench and press Delete; that only removes the registry entry.

Privacy and safety: attaching does not read or transmit content. AICodingEditor reads selected content only after Send to AI or Ctrl+Enter. Text/code/data files are bounded UTF-8 context. An attached folder is a bounded explicit scan—not the global Full Project Scan preference—and skips .venv, .git, dependency folders, build output, caches, keys, credentials, and likely secrets. No attached file is executed.

Images: PNG, JPG/JPEG, GIF, and WebP are sent as visual inputs for an image-capable OpenAI or Gemini model. The limit is four images, 3 MB per image, and 12 MB total raw image data. Unsupported/binary file types are identified as skipped instead of silently uploaded.

The sample prints the formulas before entering the while loop, provides a Streamlit Stop Simulation button, and also checks ESC / Ctrl+C from the console on Windows.

import streamlit as st
import numpy as np
import matplotlib.pyplot as plt
import time
import sys

# Try to import msvcrt for keyboard detection on Windows
try:
    import msvcrt
except ImportError:
    msvcrt = None

# Set up Streamlit page configuration
st.set_page_config(page_title="Sin & Cosin Real-time Plot", layout="wide")
st.title("Real-time Sin & Cosin Waveforms")

# Print to console before entering the while loop
print("y1=sin(2*np.pi*x)")
print("y2=cosin(2*np.pi*x)")

# Also display the formula on Streamlit interface
st.write("Formula 1: $y_1 = \\sin(2\\pi x)$")
st.write("Formula 2: $y_2 = \\cos(2\\pi x)$")

# Placeholder for updating matplotlib plot
plot_placeholder = st.empty()

# Add a Stop button in Streamlit to gracefully stop and free resources
stop_button = st.button("Stop Simulation")

# Initial parameters
step_num = 0
dt = 1.0 / 36.0          # Each step is 1/36 second
sleep_time = 1.0 / (5*14.4)   # Every 1/7.2 second we update
window_size = 360        # Approx 360 points in the window

try:
    while True:
        # Force close any unclosed figures before creating a new one to save memory
        plt.close('all')

        # Check if user clicked Stop in Streamlit
        if stop_button:
            st.warning("Simulation stopped via Web UI.")
            break

        # Check for keyboard interrupts (ESC or Ctrl+C) in console
        if msvcrt and msvcrt.kbhit():
            key = msvcrt.getch()
            # ESC key is 27 (b'\x1b'), Ctrl+C key is 3 (b'\x03'), 'c'/'C' for ease
            if key in (b'\x1b', b'\x03', b'c', b'C'):
                print("Exit signal received. Stopping while loop...")
                break

        # Generate x values for the moving window of 360 points
        indices = np.arange(step_num, step_num + window_size)
        x = indices * dt
        
        # Calculate y values
        y1 = np.sin(2 * np.pi * x)
        y2 = np.cos(2 * np.pi * x)

        # Plot using Matplotlib
        fig, ax = plt.subplots(figsize=(10, 2.5))       
        ax.plot(x, y1, label="y1 = sin(2*pi*x)", color="blue", linewidth=1.5)
        ax.plot(x, y2, label="y2 = cosin(2*pi*x)", color="red", linewidth=1.5)
        
        ax.set_xlim(x[0], x[-1])
        ax.set_ylim(-1.2, 1.2)
        ax.set_xlabel("x (seconds)")
        ax.set_ylabel("y")
        ax.grid(True, linestyle="--", alpha=0.6)
        ax.legend(loc="upper right")
        
        # Display the updated figure in Streamlit
        plot_placeholder.pyplot(fig)

        # Force close the current figure to free memory immediately
        plt.close(fig)

        # Move window by 10 points (which corresponds to 10 * 1/36 second = 1/3.6 second of real-time shift)
        step_num += 10

        # Delay for 1/3.6 second
        time.sleep(sleep_time)

except KeyboardInterrupt:
    print("KeyboardInterrupt (Ctrl+C) detected in console. Exiting...")
finally:
    # Cleanup all matplotlib plots to avoid resource leakage
    plt.close('all')
    plot_placeholder.empty()
    st.info("Simulation ended and all resources have been released.")
