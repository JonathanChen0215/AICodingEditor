1. 產品範圍

AICodingEditor 是多编程语言程序碼編輯器与工作平台，整合編輯、語法突顯、Workbench、文件瀏覽器、嵌入式控制台、Python uv 虛擬環境、SQL、AI 學習專案，以及程序运行数据报告。

程序語言
Java、C、C++、C#、Python、JavaScript、PHP、Rust、Arduino、SQL。
数据庫
MS SQL Server、SQLite、MySQL CLI 工作流程。
AI
机器学习、深度学习、神经网络、電腦視覺与 NLP。
运行报告
变量、标准输入／输出、表格、图像、XLSX 与 PNG 圖表。

3. 快速開始

  1. 保持 AICodingEditor.exe 与 AICodingEditor.jar 位於同一個数据夾。
  2. 打开已保存的程序文件，按「編譯目前文件」或「运行目前文件」。
  3. 互動式程序请在下方「程序标准输入」输入数据後按送出。
  4. 需要报告与即时变量时，启用「变量／圖表 → 启用即时变量監測」，选择变量後再运行。
「編譯」只進行檢查／建置；「运行」才會实际运行程序並在原始碼同層建立 variable 报告数据夾。

5. 数据采集、Excel 报告、表格与图像

每次实际「运行」會自動建立：

<程序文件同層>/variable/<檔名_时間戳>/
  AICodingEditor_RunReport.xlsx
  console_io.txt、variables.csv、run_metadata.txt
  table/、fig/
項目采集方式結果
程序输入透過程序标准输入送出的文字。Variables 与 Console I-O 工作表。
程序输出实际运行期間的标准输出／错误。Console I-O 工作表与文字紀錄。
变量AICE_VAR name=value、name=value、name: value 或單行 JSON。Variables 工作表与 variables.csv。

6. 即时变量監測与波型圖

  1. 需要即时遥测时才按工具栏 ⌁ 启用即时变量监测；默认停用且不显示下方“变量监测”分页。
  2. 按「选择变量...」，從检测清單勾選或输入变量名稱（用逗號分隔）。未選取代表監測所有可辨識输出变量。
  3. 启用后再运行程序，右下“变量监测”分页才会出现并更新数据。
  4. 表格會显示目前值、样本數、时間与來源；選取数值变量或使用下拉選單即可显示即时波型／趋势。
  5. 程序結束後按「打开 variable 报告」可找到 XLSX、table 与 fig。

目前執行期間會保留完整的數值樣本歷程；資料列不會因週期重複而合併。

若每个周期有 360 笔、共运行 10 个周期，报表会保留 3,600 笔独立数据行。即使数值重复，也会因产生时间不同而完整保留。variable_events.csv 另保留 stdin / stdout 的长格式审计记录。

V0.1.35：uv 启动检查与 Python 套件自动修复

AICodingEditor 启动时会检查已设置的 uv 命令。如果找不到 uv，系统会自动尝试：

python -m pip install uv

如果普通安装位置没有写入权限，系统会再用 --user 重试。处理过程显示在嵌入式控制台；即使失败，编辑器仍可继续使用并显示诊断信息。可在 设置 → 首选项 → 常规 选择 Python 解释器和 uv 路径。

运行 Python 时缺少第三方 site-package

当 Python 程序出现 ModuleNotFoundError: No module named '包名称' 时，系统会弹出确认窗口并提供建议的 PyPI 包名称（例如 sklearn → scikit-learn、cv2 → opencv-python）。点击确定后，系统在当前 Python / uv 环境安装该包；安装成功后自动重新运行同一个文件。

# 会触发自动修复询问的错误示例
ModuleNotFoundError: No module named 'humanize'

发行包内的 Examples/Python/AutoInstall_Humanize_Demo.py 可立即测试此流程。

Python 标准库 import 提醒

标准库不需要也不应通过 PyPI 安装。当系统识别常见标准库的 NameError，例如 name 'json' is not defined，会提示在代码上方加入 import，而不会建议 pip install：

import json

Examples/Python/StandardLibrary_Import_Reminder_Demo.py 可用于测试；该示例故意注释了 import json。

安全机制：缺少第三方包时，AICodingEditor 一定先询问用户是否同意，才会执行安装。需要网络连接和可用的 pip；为避免无限重试，每次手动运行只会自动重跑一次。

V0.1.35：EXE 同级工具链与 Jupyter Notebook

AICodingEditor.exe 同级工具链文件夹

第一次启动时，默认的工具链公共根目录就是 AICodingEditor.exe 与 AICodingEditor.jar 所在文件夹。请将已有的第三方工具文件夹放在启动文件同一层：

Windows_Executable/
  AICodingEditor.exe
  AICodingEditor.jar
  JDK-26.0.1/        （Java 工具）
  Dev-Cpp/           （MinGW gcc/g++）
  ArduinoCLI/        （arduino-cli.exe）
  SQL_Driver/        （SQLite / MS SQL Driver 文件）
  Manuals/
  Examples/

AICodingEditor 会将实际存在的同级工具文件夹自动加入到每个编译、运行、Python、Streamlit、Jupyter 和嵌入式控制台子进程的 PATH 前端。不会修改 Windows 系统或用户的全局 PATH 注册表值，也不需要管理员权限。工具不在同级时，可在 设置 → 首选项 → 常规 → 程序语言与工具链 修改根目录。

可编辑的 .ipynb Notebook

直接打开 Jupyter .ipynb 文件即可。AICodingEditor 会用单元格文本模式显示：

# %%
# Python 代码单元格
print("Hello Jupyter")

# %% [markdown]
# Markdown 标题与说明

保存时会写回标准 .ipynb JSON 格式。当单元格文本被修改时，原有代码单元格输出会清除，避免源代码已更改却保留过期结果。

运行当前 Notebook

  1. 打开并保存 .ipynb 文件。
  2. 点击上方 Jupyter 运行，或选择 运行 → Jupyter：运行当前 Notebook（.ipynb）。
  3. 系统会检查当前选定 Python／uv 环境是否具备 jupyter、nbconvert 与 ipykernel。
  4. 若缺少套件，确认后自动安装；有 uv 时使用 uv pip install --python <interpreter> jupyter nbconvert ipykernel，否则使用 python -m pip install。
  5. AICodingEditor 会先将当前 Python 环境注册为 Jupyter kernel，再用 jupyter nbconvert --to notebook --execute --inplace 运行全部代码单元格。输出结果会写回同一个 Notebook；点击 停止 可强制终止执行。

发行包内含不需要第三方套件的 Examples/Jupyter/SinCos_Live_Notebook.ipynb 示例。

V0.1.36：Jupyter 整数计数栏位自动修复

运行 .ipynb 前，AICodingEditor 会修复 Jupyter nbformat 要求必须为 JSON 整数的字段：nbformat、nbformat_minor、每个代码单元格的 execution_count，以及每个 execute_result.execution_count。

自动修复：可转换的整数小数，例如 "execution_count": 1.0，会改为 "execution_count": 1。非整数或不合法的执行计数会改为 null；这是 Notebook Schema 合法值，且不会猜测错误的执行顺序。DataFrame、波形／图表、影像输出与其他 metadata 的一般浮点数不会被修改。

需要修复时，编辑器会先在同一文件夹建立备份：

MyNotebook.aicodingeditor-backup-YYYYMMDD_HHMMSS_SSS.ipynb

此功能可在 nbconvert 启动 Kernel 前排除 1.0 is not of type 'integer', 'null' 等验证错误。

  3. 环境变量模式只会在发送 AI 请求时，通过 System.getenv() 读取 Key；不会显示、不会填回 Key 字段，也不会写入 Java Preferences。清除本机已保存 Key 不会删除 Windows 环境变量。
  4. 可使用 Chat、Explain Code、Fix Error、Generate Code、Review Code。默认传送当前选取代码、当前文件及 Console Error；Full Project Scan 默认关闭，必须自行明确勾选。
  5. 若要修改程序，模型必须返回 AICodingEditor patch。点击 Patch Diff Preview，查看 unified diff 与左右内容比较后，再点 Apply Patch，并于第二次确认窗口明确同意。
  6. 当 AI 返回完整代码区块或完整 Patch 后，可点击 AI 面板的“建立新档...”，或选择 AI → 由 AI 结果建立新档...。另存对话框默认使用当前打开文件的文件夹／当前工作路径；选择文件名与位置后，必须先检查预览、再确认才会创建文件。若目标文件已存在，会先显示 Diff，并需要另外确认覆盖。目标扩展名为 .ipynb 或需求包含 Jupyter／Notebook 时，代码会转换为标准 Jupyter Notebook 并以单元格文字模式打开。

安全：环境变量可让 Key 不出现在源代码及 AICodingEditor 的 Java Preferences，但不是企业级 Secret Vault；同一个 Windows 帐号下运行的程序仍可能访问。请勿将 Key 提交到 Git。发行包另含 SET_API_KEYS_WINDOWS.txt 及 Windows_Executable/Verify_AI_Environment_Variables.bat；验证档只显示 SET / NOT SET，不会打印 Key。

安全流程：V0.2.37-02 不允许 AI 自动运行程序或安装套件。Patch 必须“先预览、再确认”才会更新编辑器内容；AI 生成的新文件也不会自动写入，必须选择路径、检查预览／Diff，并明确确认创建或覆盖。

V0.2.37-33：Python 示例文件夹 – Streamlit 实时 sin/cosin 波形

Python 示例文件夹会固定保留 Examples/Python/sinx_cosinx_streamlit_2.py。此示例会建立 Streamlit 网页，实时显示 sine 与 cosine 波形，维持约 360 点的移动 x 窗口，旧点从左侧移出，新点从右侧加入。

运行方式：在 AICodingEditor 打开 Examples/Python/sinx_cosinx_streamlit_2.py，然后点击 Streamlit。当前 Python / uv 环境需要具备 streamlit、numpy、matplotlib。

示例会在进入 while loop 前打印公式，并提供 Streamlit Stop Simulation 按钮；在 Windows console 中也会检查 ESC / Ctrl+C。

查看内置示例代码
import streamlit as st
import numpy as np
import matplotlib.pyplot as plt
import time
import sys

# Try to import msvcrt for keyboard detection on Windows
try:
    import msvcrt
except ImportError:
    msvcrt = None

# Set up Streamlit page configuration
st.set_page_config(page_title="Sin & Cosin Real-time Plot", layout="wide")
st.title("Real-time Sin & Cosin Waveforms")

# Print to console before entering the while loop
print("y1=sin(2*np.pi*x)")
print("y2=cosin(2*np.pi*x)")

# Also display the formula on Streamlit interface
st.write("Formula 1: $y_1 = \\sin(2\\pi x)$")
st.write("Formula 2: $y_2 = \\cos(2\\pi x)$")

# Placeholder for updating matplotlib plot
plot_placeholder = st.empty()

# Add a Stop button in Streamlit to gracefully stop and free resources
stop_button = st.button("Stop Simulation")

# Initial parameters
step_num = 0
dt = 1.0 / 36.0          # Each step is 1/36 second
sleep_time = 1.0 / (5*14.4)   # Every 1/7.2 second we update
window_size = 360        # Approx 360 points in the window

try:
    while True:
        # Force close any unclosed figures before creating a new one to save memory
        plt.close('all')

        # Check if user clicked Stop in Streamlit
        if stop_button:
            st.warning("Simulation stopped via Web UI.")
            break

        # Check for keyboard interrupts (ESC or Ctrl+C) in console
        if msvcrt and msvcrt.kbhit():
            key = msvcrt.getch()
            # ESC key is 27 (b'\x1b'), Ctrl+C key is 3 (b'\x03'), 'c'/'C' for ease
            if key in (b'\x1b', b'\x03', b'c', b'C'):
                print("Exit signal received. Stopping while loop...")
                break

        # Generate x values for the moving window of 360 points
        indices = np.arange(step_num, step_num + window_size)
        x = indices * dt
        
        # Calculate y values
        y1 = np.sin(2 * np.pi * x)
        y2 = np.cos(2 * np.pi * x)

        # Plot using Matplotlib
        fig, ax = plt.subplots(figsize=(10, 2.5))       
        ax.plot(x, y1, label="y1 = sin(2*pi*x)", color="blue", linewidth=1.5)
        ax.plot(x, y2, label="y2 = cosin(2*pi*x)", color="red", linewidth=1.5)
        
        ax.set_xlim(x[0], x[-1])
        ax.set_ylim(-1.2, 1.2)
        ax.set_xlabel("x (seconds)")
        ax.set_ylabel("y")
        ax.grid(True, linestyle="--", alpha=0.6)
        ax.legend(loc="upper right")
        
        # Display the updated figure in Streamlit
        plot_placeholder.pyplot(fig)

        # Force close the current figure to free memory immediately
        plt.close(fig)

        # Move window by 10 points (which corresponds to 10 * 1/36 second = 1/3.6 second of real-time shift)
        step_num += 10

        # Delay for 1/3.6 second
        time.sleep(sleep_time)

except KeyboardInterrupt:
    print("KeyboardInterrupt (Ctrl+C) detected in console. Exiting...")
finally:
    # Cleanup all matplotlib plots to avoid resource leakage
    plt.close('all')
    plot_placeholder.empty()
    st.info("Simulation ended and all resources have been released.")
